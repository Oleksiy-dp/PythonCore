1. Предположим, вы филателист. У вас есть три коллекции марок. На первом изображены знаки, посвященные кошкам,
 на втором - автомобили, а на третьем - слоны. Вы можете обмениваться своими коллекциями с другими филателистами, 
 но только целыми коллекциями (например, поменять машины на велосипеды), а не отдельными марками.
Также вы хотите иметь краткое описание каждой коллекции.

Подскажите, пожалуйста, наиболее подходящий способ сохранения информации о коллекциях, отметках, из которых они состоят, и их описании.

Ответ:
Данные будут хранится в следующнм виде:
ID - int (порядковый номер в каталоге)
Group - String (Типы марок -кошки, автомобили, слоны ). Смотря как будет организована база данных. 
По хорошему лучще использовать int (номера групп) и связал бы эти номера со справочником групп марок.
Name - String (Наименование марки)
Description - String (Описание)
Availability - boolean - (наличие на складе)
Rating - int (Рейтинг)



2. Предположим, у вас одинаковые оценки в коллекции с кошками, и вы хотите хранить информацию только об уникальных. 
Как бы вы сохранили эту информацию?
Ответ:
Использовал поле Rating - int (Рейтинг)
При рейтинге выше необходимого сохранять или отображать информацию
